# PowerPoint Storyteller

## When To Use

- 情報を PowerPoint スライドにまとめるとき
- 長い説明を 3 から 8 枚の構成に圧縮したいとき
- 実装内容や比較内容を、非エンジニアにも伝わる形で整理したいとき

## Rules

- 1 スライド 1 メッセージを守る
- スライドタイトルは結論先出しにする
- 箇条書きは短く、1 行で読める長さを優先する
- 技術詳細は列挙しすぎず、意思決定に必要な情報に絞る
- 前提、現状、変更点、効果の順で流れを作る

## Recommended Structure

1. タイトル
2. 背景または課題
3. 変更内容
4. 実行フロー
5. メリット
6. 注意点または次アクション

## Output Checklist

- deck 全体タイトルがある
- 各スライドに短い見出しがある
- 箇条書きは 1 から 6 個に収まる
- 読み手が次の行動を判断できる
